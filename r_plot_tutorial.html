1Introducci√≥n a plot() en R Markdown

La funci√≥n plot() en R es incre√≠blemente vers√°til y es la base para crear una gran variedad de gr√°ficos.
Cuando utilizas plot() dentro de un chunk de R en tu documento R Markdown, la gr√°fica se generar√° y se incrustar√° autom√°ticamente en el documento final, ya sea HTML, PDF o Word.

2Conceptos B√°sicos de plot()

La sintaxis m√°s b√°sica de plot() es plot(x, y), donde x e y son vectores num√©ricos. Esto crear√° un gr√°fico de dispersi√≥n por defecto.

Ejemplo 1: Gr√°fico de Dispersi√≥n Simple

```{r scatter_plot_simple} # Generar datos aleatorios x <- rnorm(100) # 100 n√∫meros de una distribuci√≥n normal y <- rnorm(100) # Crear el gr√°fico de dispersi√≥n plot(x, y, main = "Gr√°fico de Dispersi√≥n Simple", # T√≠tulo principal xlab = "Variable X", # Etiqueta del eje X ylab = "Variable Y") # Etiqueta del eje Y ```

üìö Explicaci√≥n Detallada

  • rnorm(100) genera 100 n√∫meros aleatorios de una distribuci√≥n normal est√°ndar
  • main establece el t√≠tulo principal del gr√°fico
  • xlab y ylab definen las etiquetas de los ejes

3Tipos de Gr√°ficos con el Par√°metro type

El argumento type en plot() te permite especificar c√≥mo se deben dibujar los datos. Esto convierte a plot() en una funci√≥n muy flexible para diferentes tipos de visualizaciones.

"p"

Puntos (por defecto)

"l"

L√≠neas

"b"

Puntos y l√≠neas

"c"

Solo l√≠neas de "b"

"o"

Puntos y l√≠neas superpuestas

"h"

Histograma (l√≠neas verticales)

"s"

Escalera (pasos)

"n"

No dibuja (solo ejes)

Ejemplo 2: Gr√°fico de L√≠neas

```{r line_plot} # Generar datos para una secuencia x_seq <- 1:10 y_seq <- x_seq^2 # Y es el cuadrado de X # Crear un gr√°fico de l√≠neas plot(x_seq, y_seq, type = "l", # "l" para l√≠neas main = "Gr√°fico de L√≠neas", xlab = "Valores de X", ylab = "Valores de Y") ```

Ejemplo 3: Puntos y L√≠neas

```{r points_and_lines} x_data <- 1:15 y_data <- sin(x_data) + runif(15, -0.5, 0.5) # Seno con ruido plot(x_data, y_data, type = "b", # Puntos y l√≠neas main = "Gr√°fico de Puntos y L√≠neas", xlab = "√çndice", ylab = "Valor", pch = 16, # Car√°cter para los puntos col = "blue") # Color de puntos y l√≠neas ```

4Personalizaci√≥n Avanzada de la Apariencia

plot() ofrece una gran cantidad de argumentos para personalizar la apariencia de tu gr√°fico hasta el m√°s m√≠nimo detalle.

Par√°metro Descripci√≥n Ejemplo

col Color de puntos/l√≠neas col = "red" o col = "#FF0000"
pch S√≠mbolo de los puntos (0-25) pch = 19 (c√≠rculo s√≥lido)

lty Tipo de l√≠nea lty = 2 (punteada)
las Orientaci√≥n de etiquetas las = 1 (horizontal)

Ejemplo 4: Personalizaci√≥n Avanzada

```{r advanced_plot_customization} set.seed(123) # Para reproducibilidad x_val <- seq(0, 2*pi, length.out = 50) y_val <- cos(x_val) + rnorm(50, sd = 0.1) plot(x_val, y_val, type = "p", # Puntos main = "Funci√≥n Coseno con Ruido", xlab = "√Ångulo (radianes)", ylab = "Coseno(x) + Ruido", xlim = c(0, 2*pi), # L√≠mites del eje X ylim = c(-1.5, 1.5), # L√≠mites del eje Y col = "darkgreen", # Color de los puntos pch = 19, # Tipo de punto cex = 1.2, # Tama√±o de los puntos lwd = 2, # Ancho de l√≠nea las = 1, # Orientaci√≥n de etiquetas bty = "l") # Tipo de caja # A√±adir una l√≠nea de regresi√≥n abline(lm(y_val ~ x_val), col = "red", lty = 2, lwd = 2) # A√±adir texto al gr√°fico text(x = 1, y = 1.3, labels = "Algunos puntos", col = "purple") ```

5A√±adir Elementos a Gr√°ficos Existentes

Una vez que has creado un gr√°fico con plot(), puedes a√±adir m√°s elementos utilizando funciones de "bajo nivel" como lines(), points(), abline(), text(), y legend().

Ejemplo 5: Construcci√≥n de Gr√°ficos por Capas

```{r add_elements} # Crear un lienzo vac√≠o plot(NA, xlim = c(0, 10), ylim = c(0, 100), main = "A√±adir Elementos a un Gr√°fico", xlab = "X", ylab = "Y") # A√±adir primera serie de puntos points(x = 1:5, y = (1:5)^2, col = "red", pch = 17, cex = 1.5) # A√±adir segunda serie con l√≠neas points(x = 6:10, y = 100 - (6:10)^2, col = "blue", pch = 15, cex = 1.5) lines(x = 6:10, y = 100 - (6:10)^2, col = "blue", lty = 2, lwd = 2) # A√±adir l√≠nea horizontal de referencia abline(h = 50, col = "darkgreen", lty = 3, lwd = 3) # A√±adir leyenda legend("topleft", legend = c("Serie 1", "Serie 2", "L√≠nea de Referencia"), col = c("red", "blue", "darkgreen"), pch = c(17, 15, NA), lty = c(NA, 2, 3), lwd = c(NA, 2, 3)) ```

üîß Funciones de Bajo Nivel

  • points(x, y, ...) - A√±ade puntos al gr√°fico
  • lines(x, y, ...) - A√±ade l√≠neas conectando puntos
  • abline(h = valor) - A√±ade l√≠nea horizontal
  • abline(v = valor) - A√±ade l√≠nea vertical
  • abline(a, b) - A√±ade l√≠nea con intercepto y pendiente
  • text(x, y, labels) - A√±ade texto en coordenadas espec√≠ficas
  • legend() - A√±ade leyenda al gr√°fico

6Graficaci√≥n de Funciones Matem√°ticas con curve()

La funci√≥n curve() es ideal para graficar funciones matem√°ticas de una sola variable de manera eficiente. Es especialmente √∫til cuando quieres visualizar el comportamiento de funciones matem√°ticas.

Ejemplo 6: Graficar una Funci√≥n Simple

```{r plot_function_simple} # Graficar la funci√≥n y = x^2 curve(x^2, from = -5, to = 5, main = "Gr√°fico de la Funci√≥n f(x) = x¬≤", xlab = "x", ylab = "f(x)", col = "blue", lwd = 2) ```

Ejemplo 7: M√∫ltiples Funciones

```{r plot_multiple_functions} # Graficar la funci√≥n y = sin(x) curve(sin(x), from = -2*pi, to = 2*pi, main = "Funciones Trigonom√©tricas", xlab = "x", ylab = "Valor", col = "blue", lwd = 2) # A√±adir cos(x) al mismo gr√°fico curve(cos(x), add = TRUE, col = "red", lty = 2, lwd = 2) # A√±adir tan(x) curve(tan(x), add = TRUE, col = "green", lty = 3, lwd = 2) # A√±adir leyenda legend("bottomleft", legend = c("sin(x)", "cos(x)", "tan(x)"), col = c("blue", "red", "green"), lty = c(1, 2, 3), lwd = 2) ```
‚ö†Ô∏è Importante: El argumento add = TRUE es crucial para a√±adir curvas a un gr√°fico existente. Si lo omites, R crear√° un nuevo gr√°fico para cada curve().

7Combinaci√≥n de Datos y Funciones

Una aplicaci√≥n muy poderosa es combinar datos observados con funciones te√≥ricas o ajustadas, lo cual es fundamental en an√°lisis estad√≠stico y modelado matem√°tico.

```{r data_and_function_plot} # Generar datos simulados con tendencia lineal set.seed(456) x_obs <- seq(0, 10, length.out = 30) y_obs <- 3 * x_obs + 5 + rnorm(30, sd = 5) # Crear gr√°fico de dispersi√≥n plot(x_obs, y_obs, main = "Datos Observados y L√≠nea de Tendencia", xlab = "Tiempo", ylab = "Medici√≥n", pch = 16, col = "darkorange") # Ajustar modelo lineal modelo_lineal <- lm(y_obs ~ x_obs) # Graficar la funci√≥n estimada curve(coef(modelo_lineal)[1] + coef(modelo_lineal)[2] * x, add = TRUE, col = "purple", lwd = 3, lty = 1) # A√±adir leyenda legend("topleft", legend = c("Datos Observados", "L√≠nea de Regresi√≥n"), col = c("darkorange", "purple"), pch = c(16, NA), lty = c(NA, 1), lwd = c(NA, 3)) ```

8Configuraci√≥n de Chunks en R Markdown

Al usar plot() en R Markdown, es importante conocer las opciones de chunk que controlan c√≥mo se muestran los gr√°ficos en el documento final.

üìã Opciones Principales de Chunk para Gr√°ficos

  • fig.width, fig.height - Dimensiones de la figura en pulgadas
  • fig.align - Alineaci√≥n de la figura: "left", "center", "right"
  • fig.cap - T√≠tulo de la figura
  • echo = FALSE - Oculta el c√≥digo R, solo muestra la gr√°fica
  • include = FALSE - Oculta tanto el c√≥digo como la gr√°fica
  • cache = TRUE - Guarda el resultado para evitar recalcular
```{r ejemplo_chunk_options, fig.width=8, fig.height=6, fig.align='center', fig.cap='Ejemplo de gr√°fico con opciones de chunk'} # Este chunk tiene configuraciones especiales plot(1:10, 1:10, main = "Gr√°fico con Opciones de Chunk", xlab = "X", ylab = "Y", pch = 19, col = "red", cex = 2) ```
Consejo: Usa knitr::opts_chunk$set() al inicio de tu documento para establecer opciones globales para todos los chunks.

üöÄ Mejores Pr√°cticas

  1. Nombres descriptivos para chunks: Usa nombres significativos como {r analisis_ventas} en lugar de {r chunk1}
  2. Comentarios en el c√≥digo: Documenta tu c√≥digo para que sea comprensible
  3. Consistencia visual: Mant√©n un estilo coherente en todos tus gr√°ficos
  4. Tama√±os apropiados: Ajusta fig.width y fig.height seg√∫n el contenido
  5. Colores accesibles: Usa paletas de colores que sean distinguibles para todos
  6. Etiquetas claras: Siempre incluye t√≠tulos y etiquetas de ejes descriptivos
